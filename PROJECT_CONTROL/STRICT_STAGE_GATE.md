# 严格阶段门禁

## 总规则

- 按原报告执行，不按简化步骤跳过。
- 每个阶段完成后更新状态文档。
- 每个阶段完成后 git 提交。
- 交易所固定 OKX。
- 不自动实盘下单。

## 阶段 0：项目控制

必须有：
- 原报告备份。
- OKX 实施表。
- 状态文档。
- 下一步文档。
- git 仓库。

## 阶段 1：基础工程

必须有：
- Python 包。
- config。
- tests。
- 本地依赖。
- OKX 映射。

## 阶段 2：数据层

必须有：
- 1h OKX Parquet 读取。
- 已闭合 K 线过滤。
- 时间、重复、OHLCV、成交量检查。
- 数据质量报告。

## 阶段 3：特征层

必须有：
- EMA。
- ATR。
- 突破窗口使用 `shift(1)`。
- 4h 只用已完成 bar。
- 1h/4h 对齐。
- 防未来函数测试。

## 阶段 4：策略与风控

必须有：
- long / short / reject。
- entry、SL、TP、max_hold。
- 同一 symbol 单仓。
- 逐仓。
- 单向。
- 最大 10 倍杠杆。
- 27% 停机。

## 阶段 5：成本与 OKX

必须有：
- taker 手续费。
- 滑点。
- 参与率约束。
- 资金费跨时点估算。
- OKX `instId`、`tdMode=isolated`、`clOrdId`。

## 阶段 6：回测验证

必须有：
- 432 参数网格。
- 前 75% 训练。
- 后 25% 验证。
- 训练选参。
- 验证只跑一次。
- 失败判定。
- 单币结果表。
- 组合结果表。

## 阶段 7：报告与面板

必须有：
- 交易明细。
- 单币结果表。
- 组合结果表。
- 报告摘要。
- 半自动信号 JSON。
- 本地面板。
- 无自动下单。

## 阶段 8：验收

必须有：
- 全量测试通过。
- 状态文档更新。
- 下一步文档更新。
- git 提交。
