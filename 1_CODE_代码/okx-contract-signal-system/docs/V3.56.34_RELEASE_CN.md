# OKX 合约信号系统 v3.56.34 发布说明

发布日期：2026-06-29

## 本次目标

本版本修正新策略研究中的币种覆盖规则：21个成熟OKX USDT永续继续作为用户允许交易池，但不再要求每条策略覆盖21币、18币或其他固定数量。盈利质量和稳健性是目标；单币策略同样允许进入严格漏斗。

## 研究范围策略

`config/research_universe_policy.json`升级为v2：

- 允许新候选在PnL打开前冻结1—21个币；
- 子集必须由机制、点时原始字段、协议结构或前瞻流动性规则决定；
- 禁止先测试21币后保留历史赢家；
- 禁止查看PF、胜率、收益或回撤后删除亏损币；
- 禁止利用新政策修改H22、V357既有冻结成员、参数、方向或节奏；
- 窄子集证据不得外推到未交易币种。

## 自动研究门禁

`config/research_candidates/PRE_PNL_CANDIDATE_TEMPLATE.json`和`scripts/system_check.py`同步支持候选特定子集：

- 候选必须显式写入冻结币种列表；
- 币种必须来自允许的21币池；
- 子集不得重复、不得为空、规模必须为1—21；
- 必须记录收益前锁定依据，并声明未使用历史结果挑币；
- 数据覆盖门对候选自身子集要求100%，默认最少1币；
- 样本交易不得出现未声明币种。

集中度门按子集大小调整：

- 1币：不适用“单币正贡献不超过25%”；
- 2—5币：25%只作诊断，不作机械否决；
- 6—21币：继续执行单币正贡献不超过25%。

所有规模仍必须通过时间分段、市场状态、月份、单笔与前三笔贡献、有效盈利交易数、手续费/滑点/资金费压力、未来函数、参数自由度、家族去重和冻结前向验收。

## 历史审计解释

原固定21币免费链上字段审计没有被删除或篡改：

- `NO_FREE_FIELD_REACHES_FIXED21_COVERAGE_GATE`继续否定要求18币以上统一口径的链上横截面因子；
- 它不再全局否定BTC单币、BTC/ETH双币或自然同口径小子集；
- 小子集重开必须另立收益盲协议，不能根据后来收益选择币种；
- 钱包标签历史版本、字段修订、链级与代币级口径差异仍是有效门禁。

## 现有候选

- H22保持原冻结18币合格子集及60/90天前向协议；
- V357保持原冻结成员、参数、止损和持有期；
- H27仍为只读组合观察；
- 本版本没有让任何候选晋级A级，也没有打开新候选PnL。

## 安全边界

- 系统继续只产生研究与信号，不下单、不持仓、不访问账户；
- `approved strategy version`仍为`3.56.15`；
- 严格研究身份仍为`v3.56-strict`；
- 自动晋级、自动下单和自动调参仍为禁用状态。

应用与Python包版本：`3.56.34`
