# OKX 合约信号系统 v3.56.35 发布说明

发布日期：2026-07-01

## 本次目标

修复桌面程序点击“打开面板”后浏览器显示无法连接的问题。故障发生时主信号进程仍在运行，但本地 Dashboard 的 `127.0.0.1:3001` 端口没有成功监听。

## 修复内容

- Windows 桌面端优先直接使用本地 `node.exe` 启动项目内的 Next.js CLI，绕过可能静默失败的 `npm.cmd` 启动链；
- 本地 Next.js CLI 不存在时仍保留 `npm run dev:local` 回退；
- 不再固定等待2.5秒后直接打开浏览器；
- 改为每500毫秒检查一次3001端口，确认面板服务已经监听后再打开；
- 面板进程提前退出时显示退出码；
- 20秒内仍未监听时明确提示启动超时，避免只出现浏览器“无法连接”而没有系统诊断；
- 启动阶段的15分钟闭合K线核对对单币临时网络失败自动重试3次，避免一次REST抖动直接让实时监控停止。

## 验证

- 本地 Next.js CLI 可在 `127.0.0.1:3001` 正常启动；
- `tests/test_desktop_runtime.py` 增加直接启动、npm 回退和15分钟闭合K线临时失败重试测试；
- 桌面运行回归测试通过。

## 安全边界

- 本次只修复本地面板启动与就绪检测；
- 未修改策略、信号、回测、前向验收或数据逻辑；
- 系统继续只生成信号，不下单、不持仓、不访问交易账户；
- `approved strategy version` 仍为 `3.56.15`；
- 严格研究身份仍为 `v3.56-strict`。

应用与 Python 包版本：`3.56.35`
