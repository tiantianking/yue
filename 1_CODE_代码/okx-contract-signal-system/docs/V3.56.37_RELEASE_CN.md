# OKX 合约信号系统 v3.56.37 发布说明

日期：2026-07-01

## 本次修正

本版本只做研究工具与上线运行代码的边界拆分，不改变策略参数、信号规则、飞书等级或交易逻辑。

### 上线运行链

- 新增 `scripts/runtime_check.py`，只包含部署前检查与运行健康检查；
- systemd 启动前检查、健康定时器和 Linux 安装器全部改为调用该轻量文件；
- 新增 `src/okx_signal_system/runtime_manifest.py`，运行链只读取并严格校验冻结的 approved manifest，不再导入研究包；
- 正式发布清单改为生产白名单，只包含实时信号、21币数据、风险、质量筛选、生命周期、面板、飞书、缓存、部署和必要运行文档。

### 本地研究链

以下内容继续保留在本地完整仓库，但不进入生产发布包：

- `scripts/system_check.py` 的未来函数、过拟合、封存留出、邻域、随机检验、家族去重、成本压力和失败归档；
- PRE_PNL候选模板、候选工厂、历史回测、参数搜索、研究协议、试验账本和研究测试；
- 并行前向验收管理、研究文档和失败策略管理工具。

### 信号门禁没有删除

生产运行仍保留并继续阻断不可靠信号：

- approved manifest语义、版本、参数与哈希校验；
- 只使用闭合K线、最新数据时效和21币覆盖检查；
- t时点因果信号与过期信号拦截；
- 风险校验、信号质量分级、相关性控制和候选选择；
- 重复抑制、生命周期、失效清理和飞书outbox健康；
- `SIGNAL_ONLY`、只读数据、无私钥和禁止自动下单/自动平仓检查。

V357的两个冻结候选文件和 `shadow_ensemble.yaml` 继续保留，因为桌面运行链实际加载独立前向观察；它仍不能替代正式approved strategy或进入正式通知生命周期。

## 发布边界

生产服务不得导入 `scripts/system_check.py`、候选工厂或 `okx_signal_system.backtest`。新增研究工具默认不得加入 `RELEASE_FILES.txt`；只有被运行入口或运行依赖实际引用、且通过回归验证的文件才允许进入生产白名单。

应用与Python包版本：`3.56.37`
