# OKX 合约信号系统 v3.56.10 发布说明

本版本完成上线前部署工程闭环，不改变策略门槛，不增加自动下单能力，也不伪造正式策略批准文件。

## 主要变化

- 修复 `FEISHU_ENABLED` 环境总开关未完整生效的问题；现在每次发送前都会动态检查。
- 为 `python -m okx_signal_system.scheduler` 增加正式命令行入口。
- 新增部署预检脚本，检查 SIGNAL_ONLY、安全配置、私有凭据、目录权限、飞书和 approved manifest。
- 新增运行健康检查，覆盖状态新鲜度、WebSocket、闭合 K 线、运行错误、outbox dead letter 和生产推送状态。
- 新增 Linux systemd 主服务、健康检查定时器、logrotate 和安装脚本。
- 安装脚本严格按照 `RELEASE_FILES.txt` 白名单复制，避免误带 `.env`、数据库、缓存和日志。
- 将依赖锁文件转换为 Linux 可直接解析的 UTF-8，并移除测试依赖、补齐 `websocket-client` 固定版本。
- 新增 observation 与 production 双阶段上线流程及完整中文运维清单。

## 安全原则

- observation 模式可以验证公共行情，但正式推送保持 fail-closed。
- production 模式必须有当前版本合法 approved manifest 和完整飞书配置。
- 私有 OKX API Key、Secret 和 Passphrase 必须为空。
- 升级时保留 `outputs/` 和 `logs/`。
- 服务使用低权限 `okxsignal` 用户，异常自动重启，每五分钟健康检查。

## 版本

- 应用与 Python 包：`3.56.10`
- approved strategy version：`3.56.10`
- strict research identity：`v3.56-strict`

旧 `3.56.9` manifest 会按设计失效。正式推送必须从合法 strict-research 产物重新晋级。
