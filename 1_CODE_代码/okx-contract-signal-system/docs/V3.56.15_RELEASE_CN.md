# OKX 合约信号系统 v3.56.15 发布说明

发布日期：2026-06-26

## 本次更新

### 1. 候选级数据宇宙门禁

研究候选现在可以在 `data.symbols` 中声明自己的固定币种宇宙。数据完整性、历史深度、新数据门槛和覆盖率只检查候选实际使用的币种，不再因系统中已排除的短历史币种而错误阻断研究。

默认行为保持不变：未声明候选币种时，仍检查系统配置的全部币种。

### 2. 永久失败候选自动跳过

批量候选工厂会跳过以下终止状态：

- `REJECT_*`
- `ARCHIVED_*`
- `WITHDRAWN_*`
- `FAIL_STOP_NO_RESCUE`
- `REJECT_AND_ARCHIVE_NO_RESCUE`

这样可以保留失败候选的完整审计记录，同时避免开机或定时流程重复验证已经永久淘汰的策略。

### 3. H19失败家族登记

固定板块56日排名动量 `H19_FIXED_SECTOR_56D_RANK_MOMENTUM_V1` 已完成收益前审计和校准段验证：

- 预收益表示不变性：100%；
- 与14日动量平均同向权重重合：24.45%；
- 校准基础PF：0.9511；
- 校准总收益：-10.33%；
- 校准最大回撤：37.92%；
- 验证段和锁定段未打开；
- 禁止参数救援。

该家族已登记为 `rejected_calibration_no_rescue`，用于后续候选自动去重。

## 安全边界

- 系统仍然只生成和推送信号；
- 不下单、不持仓、不读取账户余额或仓位；
- 历史研究结果不能直接晋升A级；
- 失败策略不能通过修改参数重新包装；
- 前向影子验收标准没有降低。
