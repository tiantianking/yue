# OKX 合约信号系统 v3.56.14 发布说明

本版本把既有4小时Donchian慢趋势与波动压缩影子组合纳入统一并行前向验收，使系统从两条动量变体扩展为四个错峰研究变体。

## 第三条影子轨道

- 新增 `v357_shadow_ensemble` 前向验收轨道。
- 包含 `DC_n24_t50_slow` 与 `VCB_A` 两个冻结变体。
- 自动刷新21币种4小时影子账本，并只纳入非预热、真实前向观察。
- 当前轨道与14日动量、固定4入6出共用14/30/45/60/90日治理框架。

## 独立验收口径

- 新增冻结协议 `shadow_ensemble_forward_acceptance_protocol.json`。
- 基础成本直接使用影子运行时已经写入的 `estimated_net_r`。
- 压力成本固定为基础成本的2倍。
- 每笔观察按0.5%组合风险仅用于收益和回撤归一化，不代表执行仓位建议。
- 每个变体必须独立达到60天和50条真实前向观察，不能借用其他变体的样本数量。
- 即使全部门槛通过，也只能进入人工A级复核，不自动晋级。

## 证据完整性

- 新增SQLite账本适配器，将R倍数记录转换为统一前向状态与JSON账本。
- 预热记录被永久排除。
- 每次更新记录源状态哈希、SQLite哈希和账本哈希。
- 新增连续快照哈希链；历史条目被修改时适配器拒绝继续运行。
- 候选文件和验收协议均使用固定SHA-256校验。

## 运行修复

- 修复Windows默认GBK控制台在输出中文路径时导致并行验收命令末尾报错的问题。
- 并行入口强制使用UTF-8输出。

## 安全边界

- 所有新增轨道仍为研究级/影子信号。
- 不自动下单、不自动持仓、不自动调参、不自动晋级A级。
- 正式信号清单与生产运行策略未改变。
