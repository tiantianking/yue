# OKX 合约信号系统架构

版本：v3.56.37

## 一、永久边界

系统分为两个物理发布边界：

```text
本地完整仓库
├─ research-only：候选、回测、参数试验、未来函数/过拟合门禁、失败归档
└─ production allow-list：实时数据、信号、风险、质量、生命周期、面板、飞书、运行检查
```

本地研究可以生成候选证据，并通过人工批准流程生成冻结的 approved manifest。上线系统只能读取和校验该 manifest，不能搜索参数、重跑研究、修改候选或自动晋级。

## 二、上线运行主链

```text
OKX公开行情
  -> 闭合K线校验与运行缓存
  -> 21币覆盖和时效检查
  -> 冻结approved manifest校验
  -> 固定策略参数生成候选信号
  -> 风险校验
  -> 信号质量、相关性和等级筛选
  -> 生命周期、重复与过期处理
  -> 飞书outbox / Dashboard
```

### 1. 数据层

- 只读取OKX公开行情；
- 只使用明确标记为闭合的K线生成信号；
- 15分钟数据用于正式信号，5分钟缓存用于面板尾部展示；
- 运行缓存可写，多年正式历史数据保持只读；
- 缺失、过期、未闭合、币种覆盖不足时不得发送正式信号。

### 2. 冻结参数层

`src/okx_signal_system/runtime_manifest.py`只负责：

- 定位 `outputs/runtime/approved_strategy_manifest.json`；
- 校验版本、类型、批准时间、研究语义和人工批准字段；
- 校验manifest哈希与策略参数哈希；
- 将固定参数转换为运行时 `StrategyParams`；
- 失败时关闭正式推送并回退为不可批准状态。

运行模块不得导入候选工厂、回测器或研究门禁。

### 3. 信号层

`SignalScanService`统一执行：

- 最新闭合K线边界；
- t时点信号和下一可交易时点约束；
- 趋势、突破、投票与策略固定规则；
- 信号价格、币种、时间和失效条件；
- A/B正式候选与C级观察候选分离。

### 4. 运行风险与质量门禁

以下属于上线必需门禁，不能因拆分研究工具而删除：

- 数据闭合、完整性、时效和21币覆盖；
- approved manifest与固定参数哈希；
- 信号过期、价格/时间不一致和重复候选；
- 止损距离、成本、风险额度和策略风险约束；
- 质量评分、正式等级、相关性组和候选选择；
- 生命周期状态、重复通知、失效清理和outbox失败；
- 飞书正式模式配置；
- `SIGNAL_ONLY=true`、`DATA_READ_ONLY=true`、自动平仓关闭和无私钥。

### 5. 通知与面板

- 只有满足正式等级与全部运行门禁的信号进入正式飞书生命周期；
- B级只允许摘要，C级只允许观察；
- Dashboard显示21币数据、运行健康、候选状态、正式生命周期和隔离影子；
- 已失效、终止或过期信号不得继续显示为活跃信号。

### 6. V357隔离前向观察

`shadow_ensemble.yaml`及两个V357候选文件继续进入发布包，因为桌面GUI实际加载该隔离观察服务。它：

- 只读取闭合数据；
- 写入独立状态和SQLite账本；
- 不读取approved manifest作为自身晋级依据；
- 不进入正式A/B通知、正式生命周期或参数替换；
- 不能自动下单。

## 三、本地研究链

以下只存在于本地完整仓库，不进入 `RELEASE_FILES.txt`：

- `scripts/system_check.py`中的未来函数扫描、过拟合门禁、封存历史、参数空间、家族去重、成本压力、贡献集中和失败归档；
- PRE_PNL模板、研究协议、候选工厂和候选批处理；
- 历史回测、参数网格、研究报告和研究测试；
- 前向验收管理、试验账本和失败策略索引。

研究工具可以读取运行配置用于一致口径，但生产运行工具不得反向导入研究工具。

## 四、部署检查

生产服务只调用：

```text
scripts/runtime_check.py preflight
scripts/runtime_check.py runtime
```

`preflight`检查环境、目录、飞书、冻结manifest和禁止下单边界；`runtime`检查状态时效、WebSocket、闭合补线、21币覆盖、正式推送许可和通知outbox。

## 五、禁止事项

- 不访问账户余额、持仓或私有交易接口；
- 不提交、撤销或自动关闭订单；
- 不把研究候选或影子状态冒充正式信号；
- 不在生产机器上自动搜索参数、回测或晋级；
- 不把纯研究模板、文档、测试和失败归档加入上线白名单；
- 不因追求信号频率而绕过数据、风险或质量门禁。
