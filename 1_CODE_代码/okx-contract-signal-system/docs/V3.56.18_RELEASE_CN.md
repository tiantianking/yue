# OKX 合约信号系统 v3.56.18 发布说明

发布日期：2026-06-26

## 本次修复

### 生命周期汇总不再把终止记录误计为活跃

旧汇总逻辑只看 `setup_state`。因此记录虽然已经进入 `STOP_REACHED`、`TARGET_REACHED`、`TIMEOUT_RESULT` 或 `CENSORED`，只要其 setup 仍为 `CONFIRMED`，汇总就会错误显示为活跃。

v3.56.18 改为按完整双状态判断：

- setup 为 `TRIGGERED` 或 `CONFIRMED`；
- outcome 尚未进入任何终止状态；
- 同时满足时才计入 `active`。

`terminal` 现在按记录去重统计，不会因为同一条记录同时具有终止 setup 和终止 outcome 而重复计数。

对现有两条 HYPE 历史记录验证结果：

- 总记录：2
- 活跃：0
- 终止：2
- 当前有效信号面板可见：0
