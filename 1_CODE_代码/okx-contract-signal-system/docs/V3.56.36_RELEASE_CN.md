# OKX 合约信号系统 v3.56.36 发布说明

发布日期：2026-07-01

## 本次目标

把“过拟合与未来函数零容忍”从研究文档变成新候选打开PnL前的强制系统门禁。任何风险已发现或无法排除的候选，都不得进入收益验证、前向影子或正式信号。

> **新会话永久提醒：**封存只允许打开一次，失败不得营救，通过后仍需真实前向影子。H22、V357只能依靠冻结后的真实前向证据完成最终确认。

## 新增阻断门禁

新候选现在必须同时提交并通过：

- 6—10个月历史封存段，默认8个月；
- 封存起止时间必须位于完整历史数据范围内，且封存访问次数必须为0；
- 数据快照SHA256、切分SHA256以及封存清单文件的实际哈希核验；
- 完整策略家族试验账本，当前候选必须已经在PnL前登记；
- 点时字段证据，每个输入字段必须有可用时间规则和修订策略；
- 完整代码依赖清单，候选策略和辅助文件必须全部进入扫描；
- 最大持有期、标签观察期、purge和embargo；purge不得短于完整标签观察期。

任一证据文件缺失、声明哈希与实际文件不一致、封存被打开、试验数量不一致、点时字段覆盖不完整、代码依赖缺失或purge不足，候选都会被阻断。

## 未来函数扫描增强

AST扫描新增检测：

- 通过常量变量间接传入的负向`shift`、`pct_change`和`diff`；
- `bfill`、`backfill`及`fillna(method="bfill")`；
- `merge_asof(direction="forward")`和`merge_asof(direction="nearest")`；
- 向后或双向插值；
- 原有负向位移、居中滚动、部分向前`iloc`访问和未来标签检查继续保留。

自动扫描通过只是一项必要证据，不能代替点时数据、逐信号时间戳、数据快照和完整试验账本。

## 候选模板与文档

- `PRE_PNL_CANDIDATE_TEMPLATE.json`新增封存、试验账本、点时证据、结果期限和代码依赖字段；
- `RESEARCH_ROBUSTNESS_SCREEN_CN.md`同步零容忍执行顺序；
- README和项目总体说明同步新的候选入口边界。
