## 原包结论

上传 ZIP 与随附 SHA 文件一致，压缩数据完整。与上一份完整 v3.56.7 对比后，177 个运行相关文件逐字节一致；差异仅位于发布清理规则、发布脚本、发布安全测试、源文件清单和新增操作文档。

原包新增了日志、SQLite sidecar、锁文件排除规则以及 SHA-256 sidecar 自动生成，方向正确。但发现两个发布层缺陷：

1. 发布脚本使用 `git ls-files --cached --others --exclude-standard`，会把所有未被 Git 忽略的未跟踪文件自动打包。反例验证表明，项目根目录中的 `credentials.txt` 会进入 ZIP，现有测试无法阻止该行为。
2. 操作文档将脚本实际生成的 `*.sha256` 文件错误写成 `*.zip.sha256`。

## 修复内容

- Git 环境只允许打包已跟踪文件。
- 无 Git 环境只允许打包 `RELEASE_FILES.txt` 明确列出的文件。
- 缺失发布清单、非法路径或清单列出不存在文件时 fail-closed。
- 不再对整个项目目录执行不受控递归收集。
- SHA sidecar 显式创建父目录。
- 增加“未跟踪 credentials 文件不得进入 ZIP”和“无 Git 时必须使用清单”的反例测试。
- 修正文档中的 SHA 文件名和重新打包说明。
