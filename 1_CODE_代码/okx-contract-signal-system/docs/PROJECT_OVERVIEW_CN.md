# OKX 合约信号系统项目总体说明

当前版本：v3.56.20

最近更新：2026-06-26

## 一、项目目标

本项目是一个本地运行的 OKX USDT 永续合约研究与信号观察系统。它只负责：

- 读取和校验市场数据；
- 研究、回测和淘汰策略候选；
- 维护独立前向影子证据；
- 生成交易信号与风险提示；
- 通过飞书推送经允许的信号；
- 展示系统、数据、信号和影子策略状态。

系统不负责自动下单、持仓管理、自动平仓或账户控制。发布边界始终是 `SIGNAL_ONLY`。

## 二、固定研究边界

### 数据范围

- 交易所：仅 OKX；
- 标的：21 个 USDT 永续合约；
- 主要本地数据：5 分钟、15 分钟、1 小时及由闭合数据严格重采样得到的 4 小时数据；
- 补充数据：已经本地保存并通过语义审计的资金费、标记价、指数价等同交易所数据；
- 所有信号只能使用当时已经闭合、可观察的数据。

### 明确排除

- 不做跨交易所套利或跨交易所数据同步；
- 不依赖连续 L2 数据；
- 不把现货—永续难以真实捕捉的瞬时价差作为主要策略；
- 不用退市、解锁、安全事件等稀少边际事件凑候选；
- 不因没有候选而降低未来函数、过拟合、成本或集中度门槛。

## 三、系统结构

### 1. 数据层

负责下载、续接、去重、缺口检查、闭合状态检查、严格重采样和本地缓存。数据异常时不得生成正式信号。

### 2. 策略研究层

研究流程不是直接看收益，而是按固定漏斗执行：

1. 机制、付款者和唯一方向审计；
2. 与失败家族和现有策略去重；
3. 数据可行性、表示不变性和测量语义检查；
4. 参数自由度和未来函数扫描；
5. 随机时间安慰剂、方向反转和延迟成交检验；
6. 参数邻域盈利平台检查；
7. 滚动验证、成本压力和贡献集中度检查；
8. 加入现有影子组合后的增量价值检查；
9. 通过后才进入独立前向影子。

任何阻断项失败即归档，不允许通过事后调参、删币、改方向或加过滤器营救。

### 3. 前向影子层

当前持续观察的主要研究影子包括：

- 14 日横截面动量原始版；
- 4 入 6 出迟滞版；
- 已登记的慢速 Donchian 与波动压缩影子组合。

这些都属于 `RESEARCH_SHADOW`，不是正式 A 级策略，不自动下单。

### 4. 信号质量与生命周期层

负责：

- 检查信号价格与真实币种是否一致；
- 过滤过期、重复、终止和逻辑错误的信号；
- 记录信号生成、推送、观察和关闭过程；
- 防止历史终止信号继续显示为活跃信号。

### 5. 风控层

风险参数只作为信号建议，包括成本、波动、止损距离和杠杆建议。杠杆建议必须与策略波动和止损逻辑一致，不能代替真实账户风控。

### 6. 飞书与面板层

飞书只推送允许推送的信号和状态；面板展示 21 币种覆盖、最新数据、活跃信号、研究影子和系统健康状态。研究影子必须与正式信号分区显示。

## 四、策略等级

- `FORMAL_A`：只有完整历史门禁和冻结前向验收全部通过后才可能获得；
- `RESEARCH_SHADOW`：研究级前向观察，不自动下单，不冒充 A 级；
- `REJECT_AND_ARCHIVE_NO_RESCUE`：已失败并永久归档，不得换名字或调参数重新使用；
- `PRE_PNL`：尚未打开收益，只允许做结构、数据和因果性审计。

当前没有一个新策略被正式批准为 A 级。现有动量影子仍需满足至少 60 天、优先 90 天以及至少 50 次完全前瞻换仓等冻结验收条件。

## 六、关键验收标准

正式前向验收至少包括：

- 至少 60 天，优先 90 天；
- 至少 50 次完全前瞻换仓；
- 计入手续费、滑点和资金费；
- PF 至少 1.10；
- 压力成本 PF 至少 1.03；
- 最大回撤不超过 15%；
- 单一币种正收益贡献不超过 25%；
- 不能依赖单月或少数交易盈利；
- 不得存在未来函数或未闭合数据；
- 不得在看到结果后修改冻结规则。

## 七、失败策略归档

每个失败策略至少保存：

- `failure_summary.json`；

失败策略不得只留在项目内部输出目录，也不得删除原始失败证据。

## 八、改动、版本与 GitHub 制度

任何代码、配置、运行逻辑或研究门禁改动都必须同时完成：

1. 更新版本号；
2. 更新本总体说明中受影响的内容和当前版本；
3. 新增对应版本发布说明；
4. 更新发布文件清单；
5. 运行完整测试与源码审计；
6. 构建发布包；
7. 提交本地 Git；
8. 推送到 GitHub `origin/master`；
9. 确认本地 `HEAD` 与上游一致。

未推送到 GitHub 的代码改动只能称为“本地已提交、远端未同步”，不能称为完成。

详细制度见 `docs/CHANGE_CONTROL_POLICY_CN.md`。

## 九、主要目录

- `src/okx_signal_system/`：系统核心代码；
- `scripts/`：审计、研究、数据和发布脚本；
- `config/`：系统、风险、研究候选和影子配置；
- `tests/`：自动化测试；
- `outputs/`：运行状态和本地生成工件；
- `docs/`：总体说明、制度、部署与版本说明；
- `dashboard/`：本地状态面板；
- 桌面 `失败策略/`：用户可直接查看的失败策略永久归档。

## 十、日常使用原则

用户日常只需要启动信号系统并观察飞书与面板。研究和系统修改必须遵守冻结门禁；没有合适候选时继续寻找，但不能用重复机制、不可交易机制或低质量事件候选凑数量。
