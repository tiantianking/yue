# OKX 合约信号系统 v3.56.31 发布说明

发布日期：2026-06-28

## 发布目的

本版本只修正 V357 的机器研究身份，使系统登记与已完成的点时动态交易宇宙审计一致。它不改变 V357 的冻结候选规则、前向账本、信号生成、飞书推送、风险建议或任何下单边界。

## 审计结论

V357 原有慢速 Donchian 与波动压缩两个成员、全部参数、止损、持有期、成本和每笔风险归一化保持不变；只将今天仍存活的固定成熟币名单替换为因果点时动态币池。

动态面板每个4小时决策点要求：

- 至少连续180根闭合4小时K线；
- 过去84根 `vol_quote` 全部严格有效；
- 按过去84根闭合成交额选择前18币；
- 前12币用于原有相对强弱与市场广度；
- 必须存在下一根4小时开盘；
- 不使用未来上市、退市或存活信息。

共同896个日频统计期内，动态面板覆盖154个历史合约并完成1,896笔交易：

- 基础成本：PF 1.0552，总收益 6.78%，最大回撤 35.06%；
- 压力成本：PF 0.9386，总收益 -32.36%，最大回撤 42.47%；
- 最大单币正贡献20.79%，最大单月正贡献18.17%，集中度均通过；
- 中间阶段基础PF 0.8534、压力PF 0.7622。

正式决定：

`V357_SURVIVORSHIP_DEPENDENT_HISTORICAL_SUPPORT_REJECTED_NO_RESCUE`

## 机器门禁变更

`config/research_family_registry.json` 中 `4h_donchian_volatility_compression` 的状态由：

`shadow_reference`

改为：

`historical_support_rejected_survivorship_dependent_forward_observation_only`

同时登记 V357 两个候选ID与两个成员别名，防止未来通过改名重新引用旧固定币历史证据。

新增自动测试确认：

- V357 状态必须保持为历史支持已拒绝；
- 冻结候选别名必须在注册表中；
- 基础PF 1.0552与压力PF 0.9386失败证据不得丢失；
- 参数、成员、宇宙、日期和成本营救必须明确禁止。

以下内容没有改变：

- V357 两个冻结候选文件及其 SHA-256；
- V357 前向验收协议；
- 已登记前向观察、SQLite账本和状态文件；

- 飞书推送规则；
- 杠杆、账户、持仓和下单逻辑；
- `SIGNAL_ONLY` 发布边界。
