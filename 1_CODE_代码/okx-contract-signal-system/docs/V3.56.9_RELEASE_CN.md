# OKX 合约信号系统 v3.56.9 发布说明

## 发布目标

本版本在不加入自动下单、账户操作或真实持仓控制能力的前提下，为正式信号增加与策略保护结构一致的杠杆建议。杠杆结果只用于人工复核，不会改变信号等级、策略晋级、推送门槛或 approved manifest 安全边界。

## 主要变化

- 新增 `risk/leverage_advice.py`，输出不可执行的 `LeverageAdvice` 只读建议对象。
- 杠杆建议仅依赖入场价、止损距离、信号等级、信号分数、盈亏比和已校准质量模型，不读取 API Key、账户余额、账户权益、真实持仓、下单数量、保证金模式或交易所私有接口。
- 有效止损距离包含固定手续费和滑点缓冲，并通过标准化风险预算限制建议倍数。
- 全局建议硬上限为 5x；A- 影子信号最多 1x；B 级及以下不提供杠杆建议；质量模型缺失或未校准时退回 1x、低置信度。
- 飞书正式信号可显示杠杆建议倍数、参考保证金占比上限、标准化止损风险、限制原因和建议置信度，并明确标注仅供人工参考。
- 通知调度器在正式信号门控之后计算建议，因此该模块不能提升信号等级、增加候选数量或绕过正式策略批准。
- 修复发布安全测试发现的分发清单遗漏，并避免将建议字段伪装成实盘执行字段。

## 安全边界

- 系统仍为 `SIGNAL_ONLY`。
- 不新增下单、撤单、改单、开仓、平仓、账户余额或真实仓位访问。
- 不使用交易所最大杠杆作为推荐值。
- 不允许通过提高杠杆补偿低盈亏比、较差质量模型或较宽止损。
- 旧版 ML 和 shadow 运行时杠杆调整路径继续保持中性；只有新的确定性信号建议模块可以生成规范化人工参考值。

## 版本同步

- Python 包版本：`3.56.9`
- GUI 和启动器显示版本：由共享包版本自动显示 `v3.56.9`
- approved strategy version：`3.56.9`
- strict research identity：继续保持 `v3.56-strict`

由于 approved strategy version 与应用版本同步，旧 `3.56.8` approved manifest 会按既有规则 fail-closed。必须从合法 strict-research 产物重新晋级，不能手工修改 manifest。
