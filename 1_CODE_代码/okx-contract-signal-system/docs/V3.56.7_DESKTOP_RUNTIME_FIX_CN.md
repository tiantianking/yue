# v3.56.7 桌面运行修复说明

生成时间：2026-06-18
对应提交：`ecb40bd4818530f5b65dd476e244932900b19f39`

## 修复原因

按桌面快捷方式启动本地系统后，真实运行检查发现两处运行级问题：

1. GUI 扫描循环持续报错：`name 'load_runtime_config' is not defined`。
   原因是 `gui.py` 的信号扫描函数使用了 `load_runtime_config()`，但该函数只在另一个局部函数里导入，扫描作用域无法访问，导致 `latest_scan_status.json` 持续写入 `status=error`。

2. Dashboard K 线接口 `/api/candles/...` 返回 500。
   原因是 Dashboard 调用 Python 读取 parquet K 线时使用了系统默认 `python`，该解释器没有安装 `pandas`，而项目自带 `LOCAL_DEPS\venv` 中依赖完整。

这两项问题都会影响桌面验收：GUI 能打开但扫描状态异常，Dashboard 能打开但图表数据接口失败。

## 修复内容

- `gui.py`
  - 将 `load_runtime_config` 提升为模块级导入，保证 GUI 信号扫描循环可直接使用运行时风险配置。

- `dashboard/src/lib/runtime-paths.ts`
  - Dashboard 优先使用 `OKX_DASHBOARD_PYTHON`。
  - 未显式指定时，自动优先选择工作区自带 Python：`LOCAL_DEPS\venv\Scripts\python.exe`。
  - 如果工作区 Python 不存在，再回退到 `PYTHON` 环境变量或系统 `python`。

- `dashboard/src/lib/runtime-paths.test.ts`
  - 增加工作区 Python 优先级测试。
  - 增加缺少工作区 Python 时的回退测试。
