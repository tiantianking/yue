# OKX 合约信号系统 v3.56.28 发布说明

发布日期：2026-06-27

## 本次目标

修复研究候选的全局失败家族去重缺口，防止候选通过改名、改写字段名称或只改变表示形式，重复进入收益前实现和回测。

本版本只加强研究门禁，不改变正式信号、A级判断、飞书推送、杠杆建议、账户、持仓或下单路径。

## 问题来源

H28下行Beta非对称和H29分散度冲击在收益前审计完成后，才被发现分别重复已归档的MC02和MC05。虽然两项均未打开PnL，但说明原自动门禁只比较结构化 `family`，没有同时检查：

- 历史候选别名；
- 统一失败指纹；
- 由下划线连接的核心机制词；
- 数据不可用但已完成入口审计的家族。

因此，候选只要改写名称或描述，就可能绕过原结构相似度门禁。

## 实现变化

### 1. 三重自动去重

`system_check.py research` 现在同时执行：

1. **结构相似度去重**：继续比较核心信号、方向、选择方式、持有期、换仓节奏和交易宇宙；
2. **历史别名去重**：候选ID命中注册表中的历史家族ID或别名时直接失败；
3. **失败指纹去重**：候选即使改名，也会按拆分后的机制标签与失败指纹库比较。

### 2. 标签拆分

原标签提取会把 `downside_market_beta` 视为一个完整词，难以识别其中的 `downside` 和 `beta`。本版同时保留完整标签并拆分下划线子词，使改写后的候选仍能命中既有家族。

### 3. 失败指纹库扩展

统一注册表现在包含：

- 23个结构化参考、失败或影子家族；
- 历史候选别名；
- 30类失败或数据不可用指纹。

指纹覆盖：

- 横截面动量、反转、突破压缩、价格路径风险；
- 成交量、流动性、现货—永续、标记价—指数价；
- 订单流持续、吸收、跨币扩散、大额成交和成交尺寸；
- 资金费、持仓量、强平、ADL、保险基金、多空账户比；
- 期权方向流、期权波动率曲面；
- 日历季节性、跨币价格领先、宏观事件扩散、跨交易所；
- 配对收敛、支撑阻力重测、趋势回踩；
- 合约粒度和L2队列等缺少足够点时历史的数据家族。

### 4. 注册表完整性门禁

源代码检查要求注册表至少包含30类失败指纹。指纹缺失时，`source` 检查失败，避免发布一个退化的去重配置。
