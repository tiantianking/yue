# OKX 合约信号系统 v3.56.32 发布说明

发布日期：2026-06-29

## 发布目的

本版本纠正候选策略的研究范围定义。用户已明确：未来只考虑固定的21个成熟OKX USDT永续合约，不计划把策略用于所有历史或未来币种。

因此，307币点时动态宇宙审计回答的是“策略能否推广到整个历史市场”，而不是“策略能否用于用户固定的21个成熟币”。动态宇宙审计结果继续完整保留，但不再作为固定21币候选的直接淘汰门。

## 固定研究宇宙

新增：

`config/research_universe_policy.json`

该文件冻结：

- 未来研究、监控与验收范围为 `config/base.yaml` 中的21个成熟币；
- 不允许因为回测结果增加、删除或轮换币种；
- 退市或永久失去可交易性时，必须另行做事前治理决定，不能按历史收益挑替代币；
- 动态307币数据只用于外部泛化和可移植性压力测试；
- 固定21币结果不得被宣传为全市场普遍有效。

## 候选重新定义

### H22

机器状态改为：

`fixed_21_scope_candidate_forward_validation_pending`

H22仍是14日横截面动量的错开执行版本，不是独立Alpha。它继续使用冻结的18币合格子集、14日形成期、4入6出、三组错开、3日刷新、04:00 UTC入场和原成本模型。

固定21币范围内，H22没有因动态宇宙测试而正式淘汰，继续执行原验收：

- 60天且至少50次闭合前向观察做阶段判断；
- 优先90天且至少80次闭合观察做正式人工复核；
- PF、压力PF、回撤、成本、集中度和未来函数门槛保持不变。

### V357

机器状态改为：

`fixed_21_scope_candidate_forward_validation_pending`

V357两个冻结成员、EMA、Donchian、相对强弱、市场广度、成交量、ATR止损、持有期、成本和风险归一化全部保持不变。它继续在固定21币及原候选资格条件内积累真实前向证据。

V357未通过原冻结前向验收前，仍不得标为A级、正式信号或可用策略。

### H27

H27重新明确为：

`record_only_forward_diversification_observation`

H27不是第三条独立策略。它只读取H22和V357的冻结前向账本，记录相关性、共同亏损和共同回撤；不得搜索权重、改变成员或依靠历史组合结果晋级。

## 动态宇宙审计如何解释

H22动态宇宙基础/压力PF为0.9247/0.8730；V357为1.0552/0.9386。这些结果仍然有效，说明两条策略不能被描述为适用于任意历史币池，也不能直接扩展到新币、退市币或动态轮换币池。

但用户的真实未来任务是固定21币，因此这些结果被重新分类为：

`out_of_scope_generalization_stress_test`

它们是范围外泛化警告，而不是固定21币候选的淘汰证据。

## 方法学限制

固定21币是今天已知的成熟存活项目，因此旧历史表现仍然带有条件性，不能单独证明未来正期望。最终决定证据仍是从规则和币池完全冻结以后产生的60/90天真实前向记录。

本次修正不是为了用旧数据营救策略，也没有修改任何策略参数、方向、日期、成本或交易规则；只是让淘汰问题与用户真实交易范围一致。

## 不变边界

以下内容没有改变：

- H22、V357冻结规则与候选文件；
- 已产生的前向账本和哈希链；
- 前向时间、交易数、PF、压力成本、回撤和集中度门槛；
- 正式A级策略与 approved strategy version `3.56.15`；
- 飞书推送、杠杆建议、账户、持仓和下单边界；
- `SIGNAL_ONLY` 模式。
