## 发布文件来源

发布脚本采用 fail-closed 文件来源规则：

- 在 Git 仓库中，只打包 Git 已跟踪文件；未跟踪文件不会被自动纳入。
- 在没有 `.git` 的解压目录中，只打包 `RELEASE_FILES.txt` 明确列出的文件。
- `RELEASE_FILES.txt` 缺失、包含非法路径或列出不存在文件时，脚本会拒绝打包，不会回退为遍历整个目录。

新增源码或报告后，应先纳入 Git 跟踪；在无 Git 的归档目录中重新打包时，应同时审核并更新 `RELEASE_FILES.txt`。不要通过放宽忽略规则来自动收集未知文件。

## 重新打包

在项目根目录执行：

```powershell
py -3.12 scripts/build_release_zip.py --output "$env:USERPROFILE\Desktop\okx-contract-signal-system-v3.56.7-source-package-safety-final.zip" --sha256
```

该命令会生成：

- `okx-contract-signal-system-v3.56.7-source-package-safety-final.zip`
- `okx-contract-signal-system-v3.56.7-source-package-safety-final.sha256`

ZIP 内部路径使用标准 `/`，可被 Windows、Linux、macOS、CI 和 Python 标准库稳定解压。

## 校验

```powershell
Get-FileHash -Algorithm SHA256 "$env:USERPROFILE\Desktop\okx-contract-signal-system-v3.56.7-source-package-safety-final.zip"
Get-Content "$env:USERPROFILE\Desktop\okx-contract-signal-system-v3.56.7-source-package-safety-final.sha256"
```

两个 SHA-256 值必须一致。不要继续使用 SHA 文件与 ZIP 不匹配的旧包。

## 验证

建议在打包前执行：

```powershell

py -3.12 -m compileall -q src tests scripts main.py gui.py
cd dashboard
```

如本机具备完整测试依赖和数据环境，也可在项目根目录执行：

依赖真实历史数据目录 `JIAOYI_DATA_DIR` 的集成测试在未配置数据时会跳过。

## 部署

1. 解压完整源码包到目标目录。
2. 复制 `.env.example` 为 `.env`，只填写飞书和本地运行所需配置，不要写入 OKX 私有交易凭据。
3. 安装 Python 依赖：`py -3.12 -m pip install -r requirements.txt`。
4. 安装 Dashboard 依赖：在 `dashboard/` 下执行 `npm.cmd ci`。
5. 如需正式推送，必须通过 strict-research 生成合法候选并执行正式晋级，不能手工修改或伪造 approved manifest。
6. 在目标 Windows、真实 OKX WebSocket、飞书通知和本地历史数据目录环境中进行连续运行验证。
