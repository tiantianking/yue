# OKX 合约信号系统 v3.56.19 发布说明

## 本次目标

把新策略研究从“逐个完整回测”升级为“先证伪、再锁定验证”的统一漏斗，在不降低未来函数、过拟合、成本和集中度门槛的前提下，更快淘汰伪优势。

## 新增研究门禁

新候选在进入锁定历史验证前必须提交三类标准证据：

1. 随机时间安慰剂、方向反转与延迟一根K线测试；
2. 至少三个收益前声明的相邻参数，用于识别连续盈利平台和孤立尖峰；
3. 加入现有冻结影子组合后的增量价值与无明显恶化检查。

证据缺失、格式无效或任一阻断项失败，统一输出`FAIL_STOP_NO_RESCUE`并停止，不允许调参营救。

## 实现内容

- 新增`okx_signal_system.research.robustness_screen`；
- `system_check.py research`自动生成`robustness_screen.json`；
- 候选模板冻结500次随机试验、5%显著性、一根K线延迟和参数邻域要求；
- 候选工厂顺序加入伪优势、参数平台和组合增量门禁；
- 新增中文证据格式与执行协议文档；
- 已在收益前审计H20、H21并拒绝，两者均未打开PnL。

## 兼容原则

14日动量原始版与4入6出版本继续原有独立前向影子，不被追溯停用。它们已通过0、15、30、60分钟执行延迟稳健性测试；由于原冻结协议禁止结果打开后补测参数邻域，参数平台状态保持未知，不伪造通过，也不事后调参。

## 安全边界

- 系统仍为只信号模式；
- 不新增下单、持仓或账户控制；
- 不自动晋升A级；
- 不修改现有正式策略身份；
- 不改变飞书实时推送路径。
