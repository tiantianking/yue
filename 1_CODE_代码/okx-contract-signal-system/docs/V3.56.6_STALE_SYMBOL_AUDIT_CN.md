## 发现的问题

v3.56.5 新增的 Dashboard stale-symbol 运行态优先逻辑存在两个边界漏报：

1. 在线状态下，币种行只要 `status=passed` 且无错误，即使 `last_ts` 无效导致 `age_minutes=null`，也会被排除在 stale 列表之外。
2. 离线、stale 或 error 状态下，逻辑只返回旧的启动质检 `stale_symbols`；当该列表为空或过期时，当前 symbol rows 中的 failed/unknown/stale 币种会被隐藏。

这两项不会改变扫描与推送结果，但会让 Dashboard 健康度在异常边界下显示过于乐观。

## 修复

- `passed` 行必须同时具有有限的数值年龄且不超过 90 分钟，才视为非 stale。
- 非在线状态下，启动质检 stale 列表与当前运行 symbol rows 的 stale 结果取去重并集。
- 新增两个反例测试，分别覆盖缺失时间戳和离线当前失败行。

## 保持不变的边界

- approved runtime manifest 仍执行哈希和语义双重验证。
- manifest 缺失或版本不匹配时仍 fail-closed。
- daily learning 不接触正式候选。
- runtime cache、15m/5m closed backfill、outbox worker 与通知所有权未改动。
- 系统仍为 SIGNAL_ONLY，不包含下单和自动平仓开放路径。
