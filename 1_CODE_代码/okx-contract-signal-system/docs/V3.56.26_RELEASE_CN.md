# OKX 合约信号系统 v3.56.26 发布说明

## 发布目标

本版本只修正前向验收中“所有策略共用同一时间和换仓数量”的机械问题，不降低盈利、成本、回撤、集中度、防未来函数或防过拟合门槛，也不改变任何正式 A 级策略、信号方向或生产运行规则。

## 主要改动

### 1. 新增频率自适应前向样本档案

并行前向验收新增 `ForwardSampleProfile`。每条轨道可以在任何前向结果出现前冻结：

- 固定刷新间隔；
- 阶段判断所需日历天数与闭合观察数；
- 最终判断所需日历天数与闭合观察数；
- 阶段失败是否立即终止。

未配置样本档案的轨道继续使用原日频规则，因此现有14日动量、固定4入6出和 v357 影子组合的门槛不变。

### 2. 早期检查按策略节奏顺延

原14/30/45日检查点对日频策略保持不变。固定3日刷新低换手轨道改为：

- 第42天：证据链健康检查；
- 第90天：严重经济失败检查；
- 第135天：极端集中度检查。

这是把日频检查点乘以3日刷新间隔，保持相同数量级的观察机会，而不是降低样本要求。

### 3. 固定3日低换手版使用两级验收

`MOMENTUM_14D_FIXED_3D_REFRESH_HYSTERESIS6_V1` 现在按以下冻结规则运行：

- 90天且至少30次闭合刷新：只判断是否成为“前向存活候选”；
- 该阶段未通过不会立即归档，只继续积累到最终样本；
- 150天且至少50次闭合刷新：才执行最终人工复核或失败归档；
- 阶段通过和最终通过都不会自动晋级 A 级；
- 仍禁止自动下单、自动持仓和自动调参。

该调整发生时此轨道尚无完整前向刷新，未使用任何前向收益结果修改规则。

### 4. 最终样本改用闭合观察计数

最终到期判断由“已产生的换仓数”改为“已经闭合、可以完整计算收益的观察数”。未闭合的最新持仓不再被误算为完整证据。

### 5. 固定3日轨道纳入统一并行验收

`momentum_fixed_3d_refresh` 已登记到 `config/parallel_acceptance.yaml`，自动更新脚本、独立账本、独立状态和样本档案由统一并行验收流程管理。

## 保持不变的硬门槛

- 基础成本 PF 至少1.10；
- 压力成本 PF 至少1.03；
- 基础与压力最大回撤均不超过15%；
- 单一币种正收益贡献不超过25%；
- 单月和少数盈利刷新不得过度集中；
- 所有信号只使用闭合数据，禁止未来函数；
- 不允许根据前向结果修改参数、节奏或样本门槛；
- 正式 approved strategy version 仍为 `3.56.15`。
