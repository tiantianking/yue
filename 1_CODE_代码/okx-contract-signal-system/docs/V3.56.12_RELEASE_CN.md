# OKX 合约信号系统 v3.56.12 发布说明

本版本把策略研究升级为统一自动门禁，同时保持系统仅输出信号、数据只读、不执行交易。

## 自动门禁

- 使用 Python AST 自动检查负向 shift、负向 pct_change/diff、居中滚动窗口、前向 iloc 和未来标签引用。
- 使用结构化 family 签名，与动量、反转、4h Donchian 波动压缩和已归档失败候选自动去重。
- 根据 parameter_space 自动计算自由参数和组合数，最多 4 个自由参数、216 个组合。
- 自动计算单币、单月、单笔、前三笔和有效盈利交易数的贡献集中度。
- 从交易事实自动生成 baseline、1.5 倍和 2 倍成本压力结果。
- 失败候选自动归档，并使用失败哈希避免重复归档。
- 合并多年历史主体和最新运行缓存，自动判断总历史与新增数据是否达到研究门槛。

## 默认门槛

- 至少 12 个币种，覆盖率至少 80%。
- 历史深度至少 365 天，缺口率不超过 2%。
- 首次研究满足总历史即可；记录研究截止点后，下一轮默认至少新增 30 天闭合 15m 数据。
- 验证交易数至少 80；单币正贡献不超过 25%，单月不超过 35%，单笔不超过 25%，前三笔不超过 50%，有效正贡献交易数至少 10。

## 精简与安全

- 继续只使用 scripts/system_check.py 一个统一入口。
- 删除一处重复的运行缓存文件命名实现。
- 正式发布继续排除旧下单监控、旧 ML、重复通知、Streamlit、每日自动学习及旧兼容检查入口。
- 不新增后台研究进程、数据库、跨交易所数据或自动参数晋升。
- SIGNAL_ONLY、数据只读、禁止自动下单和禁止自动平仓保持不变。
