# OKX 合约信号系统 v3.56.29 发布说明

发布日期：2026-06-28

## 本次目标

修正H22在机器研究家族注册表中的过时状态，防止动态交易宇宙存活者偏差审计已经失败后，系统仍把H22识别为“历史支持候选”。

本版本只修正研究门禁中的证据等级，不改变正式信号、H22冻结参数、A级判断、飞书推送、杠杆建议、账户、持仓或下单路径。

## 证据来源

完整OKX点时动态宇宙已通过最终验收：

- OKX官方日包按北京时间自然日覆盖2023-07-01至2026-06-16，共1082天；
- 4小时Parquet共1,203,424行，覆盖307个历史USDT永续；
- 1082个日文件逐个重算SHA-256；
- 缺日、重复“合约+时间”、240分钟完整性、OHLCV合法性和数据清单一致性均通过；
- 原始一分钟压缩包仅流式处理，不长期保存。

收益前冻结的动态宇宙政策为：连续85根闭合4小时K线、按过去84根成交额选择当时前18名、固定排除USDC和XAUT，同时保持H22的14日形成期、4入6出、三组错开、3日刷新、04:00 UTC入场和0.4总敞口不变。

## 审计结果

- PF 1.0970；

- 压力成本PF 0.9863、总收益-3.48%、最大回撤-16.87%。

- 基础成本PF 0.9247、胜率48.38%、盈亏比0.9867、总收益-26.74%、最大回撤-38.93%；

- 最大单币正贡献占比11.36%，最大单月正贡献占比16.28%，集中度门槛通过，因此失败不是少数样本造成。

正式决定：

`SURVIVORSHIP_DEPENDENT_HISTORICAL_SUPPORT_REJECTED_NO_RESCUE`

## 实现变化

`config/research_family_registry.json` 中H22家族状态由：

`historically_supported_forward_shadow_only`

改为：

`historical_support_rejected_survivorship_dependent_forward_observation_only`

注册表同时写入失败原因和禁止营救边界。后续自动研究流程不得：

- 把固定存活币旧历史结果作为H22晋级证据；
- 把H22换名后登记为新的独立Alpha；
- 通过改变动态币种数量、成交额窗口、上市年龄、退市处理、形成期、4入6出、刷新节奏或多空比例营救；
- 因本次版本发布自动把现有前向影子标为A级。

H22现有冻结前向影子不删除，只允许继续记录今天成熟固定池的真实前向表现。V357和H27状态不受本版本自动影响。
