# OKX 合约信号系统 v3.56.8 发布说明

## 发布范围

本版本完成桌面运行链、Dashboard Python 启动兼容、4h 影子研究组合、本地历史回补工具和发布元数据的统一收口。

## 主要更新

- 新增冻结的 4h Donchian + 波动压缩影子研究组合，只读取闭合 15m K 线，严格重采样为完整 4h K 线。
- 影子组合使用独立 `outputs/shadow_ensemble.sqlite3` 和 `outputs/shadow_ensemble_status.json`，不进入正式 lifecycle、notification outbox、approved manifest、账户或订单模块。
- GUI 和 Dashboard 新增独立 `SHADOW A-` 观察区，影子异常不会阻断正式实时信号主链。
- Dashboard 调用 Python 时支持 `py -3.x` 启动器参数和带空格的解释器路径；GUI 启动 Dashboard 时显式传递当前 Python 解释器并移除不安全的 TLS 环境覆盖。
- 新增 `scripts/backfill_5m_history_range.py`，用于按目标时间范围回补、合并和验证闭合 5m 历史数据。
- 发布清单补齐影子候选、配置、源码、测试、检查脚本、回补脚本和本说明。

## 版本与正式门控

- 应用版本、Python 包版本、GUI/启动器显示版本和 approved strategy version 统一提升到 `3.56.8`。
- strict research identity 仍保持 `v3.56-strict`。
- 由于 approved strategy version 同步提升，旧 `3.56.7` approved manifest 会按既有规则 fail-closed；必须从合法 strict-research 产物重新晋级，不能手工修改 manifest。
- 本版本没有把影子组合接入正式推送决策，也没有降低 approved manifest、闭合 K 线或 outbox 的安全门槛。
