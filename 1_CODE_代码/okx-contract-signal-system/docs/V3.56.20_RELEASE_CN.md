# OKX 合约信号系统 v3.56.20 发布说明

## 本次目标

建立项目级的长期维护制度：提供统一总体说明，强制代码与文档、版本和发布记录同步，并把失败策略稳定归档到桌面失败策略文件夹。

## 新增内容

- 新增 `docs/PROJECT_OVERVIEW_CN.md`，统一说明项目目标、边界、架构、研究流程、策略等级、当前状态和日常使用原则；
- 新增 `docs/CHANGE_CONTROL_POLICY_CN.md`，冻结代码改动、版本、测试、失败归档和远端同步规则；
- 新增 `scripts/check_change_governance.py`，检查版本一致性、总体说明、当前发布说明、发布清单、桌面失败归档和跟踪分支同步状态；
- 新增 `scripts/refresh_failure_archive.py`，从研究状态文件复制失败证据到桌面 `失败策略` 文件夹，并生成中文失败说明和自动归档索引；
- 新增 `CHECK_REMOTE_SYNC.cmd`，用于远端同步后的最终一致性验收；
- README 增加总体说明、治理制度和失败归档入口。

## 失败归档增强

统一研究门禁的自动失败归档现在：

- 默认固定指向桌面 `失败策略`；
- 自动创建归档根目录；
- 复制 `robustness_screen.json`；
- 自动生成中文 `失败说明.md`；
- 保留原候选、研究报告、交易、成本和组合证据；
- 不删除旧归档，不允许失败策略事后营救。

## 发布门禁增强

源码审计新增：

- 项目总体说明必须存在且版本与应用一致；
- 改动治理制度必须存在；
- 当前版本发布说明必须存在。

行为代码发生变化时，治理脚本要求总体说明、当前版本发布说明和三个版本来源同时更新，否则检查失败。

## 当前归档状态

H20 与 H21 已位于桌面失败策略目录。刷新脚本会补充对应中文结果、失败说明和自动索引。

## 安全边界

- 仍为只信号系统；
- 不新增下单、持仓或账户控制；
- 不改变现有研究影子等级；
- 不自动晋升 A 级；
- 不读取或修改 GitHub 凭证；
- 远端未同步时必须明确标记为未完成。
