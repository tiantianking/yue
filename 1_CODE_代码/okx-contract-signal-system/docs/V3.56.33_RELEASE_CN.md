# OKX 合约信号系统 v3.56.33 发布说明

发布日期：2026-06-29

## 发布目的

根据用户指令，将当前研究重点明确冻结为：

1. H22；
2. V357；
3. H27组合分散观察。

H22与V357是当前两个固定21币条件候选。H27不是独立策略，只在两条候选账本均已更新后读取它们的前向结果。

## 观察优先级

`config/research_universe_policy.json` 新增机器可读优先级：

- `primary_candidates`: `H22`, `V357`；
- `update_order`: `H22`, `V357`, `H27`；
- H27必须等待H22与V357两条源账本均为当前状态；
- 新机制搜索继续作为次级并行轨道；
- 优先观察不得成为修改参数、规则、方向、成本或日期的理由。

`config/parallel_acceptance.yaml` 同步调整执行顺序：

- 第一轨道：H22三组错开3日刷新；
- 第二轨道：V357 Donchian/波动压缩组合；
- 原14日动量与固定3日刷新保留为次级参考轨道。

## 当前状态边界

H22与V357仍为：

`fixed_21_scope_candidate_forward_validation_pending`

这表示：

- 尚未通过正式验收；
- 尚未标为A级；
- 尚未进入正式信号准入；
- 必须继续完成各自冻结的时间、样本数、PF、压力成本、回撤和集中度门槛。

H27保持：

`record_only_forward_diversification_observation`

它不得搜索组合权重、调整成员或把组合观察包装成第三条独立Alpha。

## 自动测试

新增回归测试，固定以下机器行为：

- 并行验收前两条必须依次为H22、V357；
- 研究政策中的主候选必须为H22、V357；
- 更新顺序必须为H22、V357、H27；
- H27只能在两条源账本更新后运行；
- 优先级不能放宽任何冻结规则。

## 不变边界

本版本没有改变：

- H22或V357策略参数；
- 选币规则、方向、刷新周期、入场时点；
- 手续费、滑点、资金费与压力成本；
- 前向验收门槛；
- 正式A级策略与 approved strategy version `3.56.15`；
- 飞书正式推送、杠杆建议、账户、持仓和下单边界；
- `SIGNAL_ONLY` 模式。
