# 第十三轮漏斗：参与者约束、库存对冲与合约制度

时间：2026-06-27

状态：**完成，9个方向全部在收益前淘汰，没有打开PnL。**

## 本轮为什么做这三条线

H22、V357、H27只负责已有候选的前向证据，不能代替新策略研究。本轮因此并行检查：

1. 参与者资产负债表、保证金和抵押品约束；
2. 做市商或Dealer库存对冲；
3. 合约规则、风险档位与市场制度。

这些机制在经济上都真实存在，但“机制真实”不等于“现有字段可以形成因果、可回测、方向唯一的永续策略”。

本轮只检查字段语义、点时历史、付款者、方向与失败家族重复。没有读取未来收益、PnL、验证集或密封留出集。

## 一、参与者约束

### T13-C01 风险档位边界挤压

设想：大仓位接近更高维护保证金档位时，波动或跨档可能迫使其减仓。

OKX当前接口可以返回BTC-USDT风险档位，共观察到99档，包含：

- 初始保证金率；
- 维护保证金率；
- 最大杠杆；
- 每档最小和最大仓位。

但它不能回答：

- 哪些账户当前位于档位边界附近；
- 这些仓位是多头还是空头；
- 过去每一天的档位规则是什么；
- 跨档后账户会减仓、补保证金还是对冲。

因此既没有点时暴露，也没有唯一方向，并重复OI杠杆和强平家族。

**结论：收益前淘汰。**

### T13-C02 抵押品折扣与折价压力

设想：抵押折扣率下降或清算惩罚提高，会降低资产作为保证金的有效价值，并引起资产负债表压力。

官方接口当前可以返回多币种抵押折扣档位，本次观察到约950条币种/档位记录。但这些都是当前状态：

- 没有历史时间戳；
- 没有账户实际持有多少该抵押品；
- 不知道用户会卖币、补充USDT、替换抵押品还是减少永续仓位；
- 当前低折扣可能只是已知风险的结果，而不是未来收益的原因。

把当前折扣表回填到过去三年会直接构成信息穿越。

**结论：收益前淘汰。**

### T13-C03 借贷额度与利率约束压力

设想：借贷成本或额度紧张时，杠杆需求被限制并产生减仓压力。

问题是：

- 当前借贷利率和额度接口没有历史时间戳；
- 另一个保证金借贷比接口仍只有180条、179天；
- 借入稳定币可能用于买入，借入币可能用于做空；
- 现货保证金借贷不能识别永续对冲腿；
- 顺势与反向两种解释都成立。

它还直接命中已经登记的借贷需求失败指纹。

**结论：收益前淘汰。**

## 二、库存与对冲

### T13-I01 做市商库存外部化

Dealer库存理论认为，做市商在库存可承受区间内通过报价内部消化订单；库存超过边界后，会在公开市场外部对冲。

这是一个真实机制，但公开数据看不到：

- Dealer实际库存；
- 成交是客户信息流还是Dealer对冲；
- 同一机构在现货、期权和永续上的其他腿；
- 内部消化什么时候结束。

仅使用主动成交或OHLCV推断库存，会重新进入已经失败的订单流延续、吸收、大单和L2家族。

**结论：收益前淘汰。**

### T13-I02 资金费感知的库存再平衡

近期研究表明，永续做市商持仓不仅承担价格风险，也承担资金费现金流，因此资金费会影响最优库存控制。

但对我们的信号系统而言：

- 看不到做市商库存符号和大小；
- 大户比不能确认谁是做市商；
- 最终可观察变量仍退化为资金费和Premium；
- 这是做市控制问题，不是唯一方向的公开Alpha。

因此它重复已关闭的资金费和持仓比家族。

**结论：收益前淘汰。**

### T13-I03 指数成分对冲失衡

OKX的BTC-USDT指数当前由多个交易所价格加权组成。本次接口返回的成分包括Coinbase、Bitget、Binance、OKX和Bybit。

但：

- 只有当前成分与当前权重；
- 没有历史成分权重版本；
- 机制明确依赖跨交易所；
- 调整方向还取决于旧权重、新权重和已有库存；
- 成分调整属于事件，而不是稳定重复的合约信号。

这违反单OKX数据边界，并重复跨交易所共识和现货—永续相对价格家族。

**结论：入口淘汰。**

## 三、合约制度

### T13-S01 交易粒度与规则摩擦补偿

当前合约接口能够提供tick、lot、最小数量、杠杆和部分持仓限制字段。

但：

- 只有当前规则；
- 没有历史tick、lot、最小数量和杠杆规则版本；
- 粗tick可能提高点差，也可能增加挂单深度；
- 市场微观结构效应不能确定唯一做多或做空方向。

该方向已经命中合约粒度与流动性失败指纹。

**结论：收益前淘汰。**

### T13-S02 平台持仓上限饱和压力

设想：平台或账户接近持仓/OI上限时，新增风险受限，从而产生不对称平仓压力。

现有字段无法提供：

- 过去的持仓上限；
- 当前使用率；
- 被限制的是多头还是空头需求；
- 参与者是否转移账户、抵押品或交易场所。

部分平台OI上限字段当前还是空值。因此不能形成可验证信号。

**结论：收益前淘汰。**

### T13-S03 Premium指数钳制与规则压力

Premium历史可以按页向前读取，这一点数据上可行。但它的经济身份没有变：

- 永续相对指数的Premium；
- 标记价—指数价偏离；
- 资金费对贵侧持仓的成本；
- 收敛或拥挤反转。

这些已经分别属于现货—永续、标记—指数和资金费失败家族。改叫“合约制度压力”不能产生新机制。

**结论：全局去重门直接淘汰。**

## 四、一级研究证据

- `arXiv:2102.04591`证明保证金、杠杆和强平风险确实重要，但研究目标是估算强平概率与合理保证金，不是从公开风险档位获得唯一横截面方向；
- `arXiv:2106.06974`说明Dealer会在库存超过内部消化区间后外部对冲，但实现这种信号必须观察实际Dealer库存或可识别对冲流；
- `arXiv:2605.06405`说明资金费影响永续做市库存控制，但必须知道做市商自身库存状态，不能转化为公开单方向Alpha；
- `arXiv:2212.06888`支持Premium和资金费收敛的理论机制，但这正是项目已经关闭的家族；
- `arXiv:1009.2329`说明tick size会影响微观结构和价格统计性质，但没有给出唯一收益方向，而且因果验证必须有历史规则版本；
- `arXiv:2605.29309`说明抵押与保证金分割可以维持跨产品Carry差，但依赖ETF、期权、CME等外部市场，不符合本项目边界。

## 禁止营救

- 将当前风险档位、抵押折扣、借贷额度或合约规则回填到历史；
- 用OHLCV或聚合成交猜测Dealer库存；
- 将当前指数成分权重假设为历史权重；
- 把Premium、资金费或标记—指数收敛改名为制度策略；
- 下载跨交易所指数成分历史；
- 用规则变更、ADL、保险基金或强平稀有事件做候选；
- 打开PnL后再决定应顺势还是反向。

## 最终判断

参与者约束、库存对冲和制度规则都是真实市场机制，但在当前冻结的数据边界内，没有一个同时满足：

- 可观察的点时暴露；
- 明确付款者；
- 唯一交易方向；
- 多年历史；
- 不重复失败家族。

所以本轮不能写回测代码，也不能为了“继续研究”而强行制造候选。
